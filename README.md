# ユーザーフィードバック・リアクション部品 (Feedback Widget)

Google サイトなどの社内ポータル記事に埋め込んで使用する、macOS コントロールセンター風の洗練されたリアクションウィジェットです。

> **⚠️ Notice / 注意事項**
> 本リポジトリのソースコードはオープンソース（MIT License）として公開していますが、デプロイ済みのエンドポイントは、特定の社内Googleアカウントによる認証が必須となるようアクセス制御（ドメイン制限）が設定されています。
> そのため、外部のWebサイトやブログ等に埋め込んでも、権限のないユーザーは利用できません（専用のエラーUIが表示されます）。ご自身の環境で利用する場合は、本リポジトリをフォークし、ご自身のFirebase環境にデプロイしてご使用ください。

## ✨ 特徴
* **モダンなデザイン**: macOS の UI に着想を得た、美しく視認性の高いデザイン。
* **完全なレスポンシブ**: スマホ表示でもスクロールバーが出ず、どんな画面サイズにも最適化して表示されます。
* **インタラクティブなUI**: 未ログインユーザーの好奇心を刺激する「チラ見せブラー効果」など、細部までこだわったUX。
* **リアルタイム集計**: Firebase Firestore をバックエンドに使用し、全ユーザーのクリックがリアルタイムに反映されます。
* **セキュアな設計**: ログイン認証とバックエンドのルールにより、許可されたユーザー以外のデータへのアクセスを完全にブロックします。

## 🛠 技術スタック
* **Frontend**: React 18, Tailwind CSS, Babel (CDN経由で動作、ビルド不要)
* **Backend**: Firebase (Authentication: Google認証, Firestore: リアルタイムデータベース)
* **Security**: Firebase App Check (reCAPTCHA v3)
* **Hosting**: Firebase Hosting

## 💻 開発・デプロイ手順（フォークして使う場合）

本プロジェクトはセキュリティ対策のため、APIキーなどの設定情報を `config.js` に分離し、Gitの管理下から外しています（`.gitignore` 適用済）。

1. リポジトリをクローン後、ルートディレクトリにある `config.example.js` をコピーし、ファイル名を **`config.js`** に変更します。
2. `config.js` 内の `YOUR_***` の部分を、ご自身のFirebaseプロジェクトの設定値、および reCAPTCHA v3 のサイトキーに書き換えます。
3. `index.html` 内の `ALLOWED_DOMAINS` を、許可したいGoogle Workspaceのメールドメインに変更します。
4. 以下のコマンドで Firebase にデプロイします。
   ```bash
   firebase deploy --only hosting
   ```

## 🚀 社内での設置方法（管理者向け）

### 埋め込み方法
Google サイトの編集画面にて、以下のURLを直接埋め込んでください。（`[記事固有のID]`、`[記事のタイトル]` を書き換えます）

```text
https://[あなたのFirebase Hostingドメイン]/?pid=[記事固有のID]&ptitle=[記事のタイトル]
```

### 【重要】Googleサイトでの設置のコツ
* 編集画面の「埋め込み」メニューから **「URL」タブ** を選択し、上記のURLを直接貼り付けます。
* 挿入後、ウィジェットの**横幅を PC 画面の 1/3 程度**に狭めてください（スマホ表示時に押しつぶされてスクロールバーが出るのを防ぐため）。
* 縦幅は少し余裕を持って広めにドラッグしておくと安全です。

## 📊 管理者モード（集計・CSV出力）
設置したURLの末尾に `&mode=admin` （または `?mode=admin`）を追加してアクセスすると、管理者専用の集計画面が開きます。

**例:**
`https://[あなたのFirebase Hostingドメイン]/?mode=admin`

## 🔒 セキュリティ設定の概要
本アプリを安全に運用するため、以下の多層的なセキュリティ対策を行っています。

1. **Firebase Authentication ＆ ドメイン制限**
   Googleアカウントによるログインを必須とし、さらにフロントエンドおよびバックエンド（Firestore セキュリティルール）の両方で、許可されたメールドメインを持つユーザーのみ読み書きを許可しています。
2. **Firebase App Check**
   reCAPTCHA v3 を導入し、不正なクライアントやボットからのバックエンドへのアクセスを遮断しています。
3. **秘匿情報の分離**
   APIキーなどの環境設定を `config.js` に切り出し、リポジトリ上にはテンプレートのみを公開しています。
4. **キャッシュ制御によるUX保護**
   Firebase Hosting のヘッダー設定でキャッシュを無効化し、常に最新のセキュリティロジックとUIがユーザーに提供されるようにしています。
